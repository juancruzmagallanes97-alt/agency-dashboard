### Tailwind CSS v4 Syntax Contract

All CSS must use v4 syntax. No `tailwind.config.*` file. Tokens defined in `app/globals.css` using CSS custom properties. Tailwind utility classes compose on top of those tokens.

```css
/* globals.css — correct v4 entry */
@import "tailwindcss";
```

Custom tokens are defined under `:root` using `--` prefixed CSS properties. Tailwind v4 can consume them via `theme()` or directly as CSS variables.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, dot indicators, inline separators, badge internal padding |
| sm | 8px | Nav item vertical padding, badge horizontal padding, compact element spacing, chip padding |
| md-compact | 12px | Nav item horizontal padding (expanded state), tight component internals such as input horizontal padding | <!-- exception: compact horizontal contexts -->
| md | 16px | Card internal padding fallback, form field gaps, table header/body cell padding, settings row padding |
| md-card | 20px | Card default padding, content breathing room, AlertRow horizontal padding | <!-- exception: card/panel standard -->
| lg | 24px | Section padding, card-to-card gaps, modal padding, login form spacing |
| xl | 32px | Page content top padding, major section breaks, button/input height |
| auth-card | 40px | Auth card horizontal padding | <!-- exception: auth layout only -->
| 2xl | 48px | Auth page vertical centering offset |
| 3xl | 64px | Full-page layout breathing room |

### Spacing Exception Rationale

The standard 8-point scale (4, 8, 16, 24, 32, 48, 64) is extended with four justified additions:

- **12px (md-compact):** Nav item horizontal padding and input padding require a value tighter than 16px but looser than 8px. 12px is a standard exception for navigation and form internals in Notion-style UIs.
- **20px (md-card):** Card default padding and AlertRow horizontal padding. 16px feels cramped; 24px is too open for compact cards. 20px is the Notion-native card breathing value.
- **36px:** Table header row height and button/input heights where 32px is too compact and 48px is too tall. Explicitly listed below as dimensional exception.
- **40px (auth-card):** Auth card horizontal padding only. 48px is too wide for a 360px card; 32px leaves insufficient margin. 40px is a one-off for the auth layout.

Non-multiples of 4 that are NOT exceptions — remapped:
- Badge padding 2px/3px → **4px**
- Badge padding 6px → **8px**
- Badge padding 10px (chips) → **8px**
- Settings row padding 14px → **16px**
- `margin-top: 2px` on sub-text → **4px**
- `margin-top: 6px` on StatBlock sub → **8px**

Other fixed dimensions (not on rhythm scale — structural):
- Sidebar width: 240px (expanded), 56px (collapsed)
- Sidebar workspace header: 52px height
- Table header row: 36px height
- Table body row: 44px height (accessibility touch target)
- Nav item height: 32px
- Input / primary button height: 36px
- Logo mark: 32×32 (login), 26×26 (sidebar)

## Typography

**Exactly 4 sizes. Exactly 2 weights.**

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Default text, table cells, card content, nav labels, form inputs, integration name, HealthBadge score |
| Label | 12px | 400 (regular) | 1.4 | Section headers in sidebar, table column headers, badge text, metadata, sub-text in StatBlock, settings description, filter chip text, "Configurar" link, form labels, unit text in MetricRow |
| Heading | 20px | 600 (semibold) | 1.25 | Page titles, card section titles, modal headings, login card heading |
| Display | 28px | 600 (semibold) | 1.2 | KPI numbers in StatBlock, health score number |

### Weight Assignment Notes

- 500 (medium) occurrences eliminated: all remapped to **400** (regular) for labels and metadata, or **600** (semibold) for interactive labels that need emphasis.
- 700 (bold) occurrences eliminated: StatBlock value and HealthBadge score weight changed from 700 → **600** (semibold). Display role at 28px at weight 600 reads as sufficiently bold.
- `font-variant-numeric: tabular-nums` is applied to Display-role numbers and MetricRow values (this is a numeric variant, not a font-weight).

### Font Size Consolidation Map

| Old size | New size | Role | Rationale |
|----------|----------|------|-----------|
| 10px (sidebar section label) | 12px | Label | 10px is below legibility threshold; Label role covers this usage |
| 11px (StatBlock label, sub-text, group headers, AlertRow meta, HealthBadge label) | 12px | Label | Consolidated; 12px remains small enough for metadata |
| 13px (workspace name, login sub, section heading inside card, nav item label, integration name) | 14px | Body | 13px is a non-standard step; Body at 14px covers all these contexts |
| 14px | 14px | Body | No change |
| 20px | 20px | Heading | No change |
| 28px | 28px | Display | No change |

Font family declaration for `globals.css`:

```css
@import url('https://fonts.googleapis.com/css2?family=Inter:wght@400;600&display=swap');

:root {
  --font-sans: 'Inter', ui-sans-serif, -apple-system, BlinkMacSystemFont, 'Segoe UI', system-ui, sans-serif;
}

body {
  font-family: var(--font-sans);
  font-size: 14px;
  line-height: 1.5;
  -webkit-font-smoothing: antialiased;
}
```

Preferred implementation: `next/font/google` with `Inter` preload (`weights: ['400', '600']`) in `layout.tsx` rather than a raw `@import`. Apply via CSS variable to body.

### Accent Reserved For

The accent color (`#2383E2`) is used ONLY on:
1. Active sidebar nav item background tint (`rgba(35, 131, 226, 0.08)`) and active nav icon/text
2. Primary CTA buttons (Login, Save Settings)
3. Link text that navigates to another page ("Ver todos", "Ver alertas")
4. Focus rings on interactive inputs (2px solid, 2px offset)

Accent is NOT used for: headings, body text, card decorations, status indicators, table rows.

## Component Contracts (DSYS-03)

All components live in `agency-dashboard/components/ui/`. TypeScript interfaces required for all props.

### Badge

Purpose: Status label for alerts, client state, connection state.

```tsx
interface BadgeProps {
  variant: 'critical' | 'warning' | 'opportunity' | 'stable' | 'neutral'
  size?: 'sm' | 'md'
  children: React.ReactNode
}
```

Visual spec:
```
size sm: padding 4px 8px, font-size 12px, font-weight 400, border-radius 4px
size md: padding 4px 8px, font-size 12px, font-weight 400, border-radius 4px
border: 1px solid (semantic border var per variant)
background: semantic bg var per variant
color: semantic text var per variant
```

Note: both Badge sizes use the same Label-role typography (12px, weight 400). Size `sm` vs `md` difference is conveyed by context/usage, not by distinct typography rules. If further visual differentiation is needed in a later phase, re-evaluate then.

### Card

Purpose: General content container. Used in Settings, Dashboard sections.

```tsx
interface CardProps {
  children: React.ReactNode
  className?: string
  noPadding?: boolean  // true for table/list cards that need flush edges
}
```

Visual spec:
```
background: var(--bg) (white card on surface-1 background)
border: 1px solid var(--border)
border-radius: 8px
padding: 20px (default), 0 (noPadding=true)
box-shadow: 0 1px 2px rgba(0,0,0,0.04)
```

### StatBlock

Purpose: Single KPI metric. Used in dashboard header row (primary focal point of the dashboard screen).

```tsx
interface StatBlockProps {
  label: string
  value: string | number
  sub?: string
  variant?: 'default' | 'critical'  // critical = red value
}
```

Visual spec:
```
container: background var(--surface-1), border: 1px solid var(--border), border-radius: 8px, padding: 20px 24px
dot indicator: 6px circle (semantic color), displayed above label
label: 12px, weight 400, text-3, uppercase, letter-spacing 0.06em
value: 28px, weight 600, text-1 (or --critical if variant=critical), line-height 1.2, margin-top: 8px
sub: 12px, weight 400, text-3, margin-top: 8px
font-variant-numeric: tabular-nums on value
```

### AlertRow

Purpose: Single alert in a list. Used in dashboard alert panel and alerts page.

```tsx
interface AlertRowProps {
  titulo: string
  clienteNombre: string
  fecha: string
  tipo: 'critico' | 'warning' | 'oportunidad'

Visual spec:
```

## Interaction Patterns

### Sidebar Collapse

- Toggle button in workspace header (PanelLeftClose / PanelLeftOpen Lucide icon)
- Collapse state persisted in `localStorage` key `sidebar_collapsed`
- Transition: `width 200ms ease`, text/labels `opacity 0→1 100ms` (delayed 80ms on expand)
- Collapsed: icons only, no labels, no section headers
- Tooltip on each nav item when collapsed: native `title` attribute (HTML) — no custom tooltip component needed in Phase 1
- Logout icon in collapsed state: `title="Salir"` tooltip via native `title` attribute

### Form Inputs (Login + Settings)

- Error state: border-color `var(--critical)` + shake animation (keyframes: translate3d -4px/4px/0, duration 400ms)
- Focus state: border-color `var(--accent)` + 2px `rgba(35,131,226,0.2)` focus ring (no outline: none)
- Loading: button disabled + spinner swap
- No inline validation during typing — validate on submit only

### Table Sorting

- Click column header to sort ascending; click again for descending; third click clears sort
- Sort direction indicated by Lucide `ArrowUp` / `ArrowDown` 12px icon adjacent to column label
- Visual change only — state managed in local React `useState`

### Filter Chips

- Click to toggle; active chips have accent styling
- Multiple chips can be active simultaneously (OR filter within same field, AND across fields)
- "Limpiar filtros" link appears (12px, text-accent) when any chip is active

## Accessibility Notes

- All form inputs must have associated `<label>` elements (not placeholder-only)
- Focus order must follow visual order in DOM
- Nav items use `<a>` (via Next.js `Link`) — correct semantic
- Active nav item must include `aria-current="page"`
- Sidebar collapse button: `aria-label="Colapsar sidebar"` / `aria-label="Expandir sidebar"`
- Logout icon (collapsed sidebar): `title="Salir"` provides tooltip; also add `aria-label="Salir"` for screen reader access
- Color is never the sole indicator — Badge always includes text label alongside color
- Minimum contrast: text on `var(--surface-1)` (#F7F7F5) at 14px → #6B6B6B (#text-2) achieves 4.58:1 (passes AA)
